# 翻译契约(所有 subagent 必须遵守)

翻译本书英文原版到中文,目标读者为 GPU / 机器学习系统工程师。**第一目标是结构守恒:不许破坏任何能影响渲染或代码运行的内容。**

## 一、逐字保留(绝对不许改动)

1. **MyST 指令与围栏**:` ```{toctree} ` / ` ```{admonition} ` / `:::` 围栏、`{ref}\`label\``、`{eval-rst}`、`{toctree}` 的 `:caption:` `:maxdepth:` 等选项行。**指令所在的那一整行原样保留。**
2. **锚点 / 标签**:形如 `(chap_gemm_basics)=` 的行原样保留;rst 的 `.. _label:` 也是。
3. **数学**:`$...$`、`$$...$$`、`\text{}`、`\top`、`\frac`、下标 `_`、`^`、LaTeX 环境等,**块内一个字符都不改**。
4. **代码块本体**:`python` / `bash` / `c` / 无标注围栏内,**除以 `#` 开头的注释行之外的所有行原样保留**——包括标识符、字符串、API 名(`tcgen05.mma`、`T.SMEMPool()`、`Tx.copy`、变量名、缩进)。
5. **`text` 代码块**:整体保留(ASCII 框图、伪代码示意、程序输出),**连注释也不译**。
6. **图片**:`![alt](../img/x.png)` 的**路径与文件名逐字保留**;只允许改写 `alt` 文字(译为中文)。
7. **链接**:Markdown `[text](url)` 与 rst 链接的 **url 部分保留**,链接文字可译。
8. **rst 指令/角色**:`.. code-block:: python`、`.. toctree::`、`:ref:` 、`.. note::` 等**指令行与选项保留**;只译其下的正文。
9. **代码块围栏标记**:` ```python ` 的语言标记保留;围栏数量、缩进、空行保持一致。

## 二、需要翻译

1. 正文叙述段落。
2. **标题文字**(保留层级 `#`/`##`/`###` 与同行锚点/`{label}=`)。
3. 列表项正文、表格单元格正文。
4. admonition / note 等指令的**正文**(指令行本身不动)。
5. **仅** `python`/`bash`/`c` 代码块内以 `#` 开头的**注释行**(把 `#` 后的英文注释译成中文,`#` 与紧跟空格保持)。
6. 图片 `alt` 文字。

## 三、文风与标点

- GPU 硬件 / API / 框架术语遵循 `GLOSSARY.md`:缩写与专有名词**保留英文**,首次出现时用半角括号补中文释义(例:"TMA(张量内存加速器)")。同一术语全文用法一致。
- 普通叙述:译成流畅、专业的中文。技术含义精确,不意译失真。
- 标点:正文用中文全角(,。:!?)。**括注英文术语用半角括号 `()`**。保留英文缩写内部的标点不动。
- 数字、单位、版本号、变量名不译。
- 不增删内容,不合并/拆分段落;段落、列表层级、缩进与原文严格对齐。
- 代码示例里的 `#` 注释译完后仍须是合法注释。

## 四、输出

- 写到指定输出路径(英文原文件保持不动)。
- 保留与原文相同的行尾风格。
- 不要添加译者注,除非 `GLOSSARY.md` 或任务特别要求。
