# 翻译契约(所有 subagent 必须遵守)

翻译本书英文原版到中文,目标读者为 GPU / 机器学习系统工程师。**第一目标是结构守恒:不许破坏任何能影响渲染或代码运行的内容。**

## 一、逐字保留(绝对不许改动)

1. **MyST 指令与围栏**:` ```{toctree} ` / ` ```{admonition} ` / `:::` 围栏、`{ref}\`label\``、`{eval-rst}`、`{toctree}` 等指令行原样保留。`{toctree}` 的 `:caption:` 可译为中文,但 `:maxdepth:` 等结构性选项原样保留。
2. **锚点 / 标签**:形如 `(chap_gemm_basics)=` 的行原样保留;rst 的 `.. _label:` 也是。
3. **数学**:`$...$`、`$$...$$`、`\text{}`、`\top`、`\frac`、下标 `_`、`^`、LaTeX 环境等,**块内一个字符都不改**。
4. **代码块本体**:`python` / `bash` / `c` / `c++` / 无标注围栏内,**除注释之外的所有可执行内容原样保留**——包括标识符、字符串、API 名(`tcgen05.mma`、`T.SMEMPool()`、`Tx.copy`、变量名、缩进)。注释可译,但不能新增或删除有效代码。
5. **`text` 代码块**:程序输出、ASCII 框图、命令输出整体保留;若 `text` 块只是概念清单或流程示意,可翻译文字,但必须保留行数、缩进和关键英文 API/硬件名。
6. **图片**:Markdown 图片的 `alt` 文字译为中文。若存在对应的 `../img/zh/...` 中文图片,路径可切换到该中文图片;否则路径与文件名逐字保留。
7. **链接**:Markdown `[text](url)` 与 rst 链接的 **url 部分保留**,链接文字可译。
8. **rst 指令/角色**:`.. code-block:: python`、`.. toctree::`、`:ref:` 、`.. note::` 等**指令行与结构性选项保留**;只译其下的正文。
9. **代码块围栏标记**:` ```python ` 的语言标记保留;围栏数量、缩进、空行保持一致。

## 二、需要翻译

1. 正文叙述段落。
2. **标题文字**(保留层级 `#`/`##`/`###` 与同行锚点/`{label}=`)。
3. 列表项正文、表格单元格正文。
4. admonition / note 等指令的**正文**(指令行本身不动)。
5. 代码块内的注释文字,包括 `#` 和 `//` 注释;译后必须仍是合法注释,且不得改变有效代码。
6. 图片 `alt` 文字。

## 三、文风与标点

- GPU 硬件 / API / 框架术语遵循 `GLOSSARY.md`:缩写与专有名词**保留英文**,首次出现时用半角括号补中文释义(例:"TMA(张量内存加速器)")。同一术语全文用法一致。
- 普通叙述:译成流畅、专业的中文。技术含义精确,不意译失真。
- 标点:正文用中文全角(，。；：！？)。**括注英文术语用半角括号 `()`**。保留英文缩写、代码、URL、数学表达内部的标点不动。
- 数字、单位、版本号、变量名不译。
- 不无故增删内容。为中文可读性可调整自然换行,但不得改变章节层级、列表层级、指令缩进、代码块数量和可引用锚点。
- 代码示例里的 `#` 注释译完后仍须是合法注释。

## 四、输出

- 写到指定输出路径(英文原文件保持不动)。
- 保留与原文相同的行尾风格。
- 不要添加译者注,除非 `GLOSSARY.md` 或任务特别要求。
