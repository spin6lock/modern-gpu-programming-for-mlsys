mbarrier：数据结构与 API

mbarrier 对象（64 位，位于共享内存）

phase
0 或 1
pending_count
剩余到达数
expected_count
总期望数
tx-count
待传字节数
phase 当前阶段（0 或 1）。完成时自动翻转。
pending_count 阶段完成前还需到达的次数。
expected_count 每个阶段期望的到达总数（由 init 设置）。
tx-count 待传输的异步字节数。由 arrive.expect_tx 增加，传输完成时由硬件减少。

阶段完成条件

pending_count == 0
&&
tx-count ≤ 0
当 所有期望线程均已到达 且 所有期望的异步字节均已传输完毕 时，一个阶段完成。 完成后屏障自动复位：phase 翻转，pending_count 复位为 expected_count，tx-count 复位为 0。

核心 API

指令角色对 mbarrier 的作用
mbarrier.init setup 用期望的到达计数初始化屏障 phase=0, pending=count, tx=0
mbarrier.arrive producer 线程在屏障上发出到达信号 pending_count -= 1
mbarrier.arrive.expect_tx producer 到达 + 声明期望的异步传输字节数 pending -= 1, tx_count += txCount
tcgen05.commit producer 张量核发出 MMA 完成信号 arrive::one on mbarrier
mbarrier.try_wait consumer 等待（挂起线程）直到阶段完成 blocks until phase done
三种到达模式： (1) mbarrier.arrive — 线程直接递减 pending_count。 (2) arrive.expect_tx — 线程到达 + 声明字节数；TMA 引擎在传输完成时自动递减 tx_count。 (3) tcgen05.commit — 张量核在 MMA 完成时自动到达。
